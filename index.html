HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content chown to the user.
  • !DOCTYPE html declaration. Tells the browser what document type it should expect when it is opened. This particular declaration lets the browser know to expect HTML on the page
  • HTML elements are case sensitive
  • HTML attribute defines additional information about an element
  • Metadata is data about our webpage that the browser uses to effectively serve our webpage users. Contains import info about accessibility, search engines, and performance
  • style element defines the CSS associated with a website
  • base element defines the base URL for a website
  • link element connects an external resource to the HTML document
  • charset means character set
  • script element embeds client-side scripts in the HTML document
  • http-equiv element sets an HTTP header for content
  • viewport element sets the default view for our webpage
  • a creates a link
  • src attribute defines the location, or source of the image file
  • Semantic HTML elements means the name of the element describes its content and the role it plays on a webpage
  • alt attribute contains a text string that describes the appearance and functionality of the image

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Inline CSS - used for styling an element directly in the HTML file
  • Internal Style Sheet - Embeds an entire style sheet directly in the HTML file using style and head
  • External CSS - the most common wya to style a webpage, allows developers to keep all their CSS rules in a separate file. Developers use a link element placed in the HTMLs head
  • Declarations contain two important components: the CSS property we want to apply and the value of the property. For example, the declaration color: blue includes the CSS property color and the value for the property, blue.
  • rel attribute specifies the relationship between the current document and the linked document resource
  • href attribute specifies the location (URL) of the external resource. Th href attribute is using a relative path. This means the location is derived from the current location. To locate the style sheet from the current location, we must look in the assets folder, hence assets/style.css
  • Write new declarations for a new CSS rule by designating a Class Attribute as the selector. This allows us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector
  • .card - the period before card designates this as s class selector, any element assigned to the class card, will gan these property assignments

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switched to it
  • Add command adds modifications in the current working branch to the staging area
  • -A flag indicates that we want to add all changes
  • Pull command to receive a branchs modifications into the local environment
  • Origin keyword indicates the source of the pull will be the GitHub repo
  • Main keyword indicates the branch, in this cases the main branch from the GitHub repo
  • Push command pushes the changes we have locally to our remote GitHub branch
  • Git command must precede every git statement
  • IDE - Integrated development environment
  • IDE is a software used by developers that combines many of the features needed by developers to do their work in a single interface GUI
  • IDE contains essential tools for code development such as a source code editor, local build automation
  • Git is a version control system that lets you manage and keep track of each version of your project.
  • Repository (repo) is a folder in which all files and their version histories are stored
  • Branch is a workspace in which you can make changes that will not affect the live site
  • Markdown (.md) is a way to write in GitHub that converts plain text to GitHub code. Sites such as Atom and Sublime Text are examples of free resources for developers using Markdown
  • Commit Changes is a saved record of a change made to a file within the repo
  • Pull Request (PR) - the way to ask for changes made to a branch to be merged into another branch that also allows for multiple users to see, discuss and review work being done
  • Merge is after a pull request is approved, the commit will be pulled in (or merged) from branch to another and then, deployed on the live site.
  • Issues is how work is tracked when using git. Issues allow users to report new tasks and content fixes, as well as allows users to track progress on a project board from beginning to end of a specific project.
  • Federalist is a platform that surely deploys a website from a GitHub repository un minutes and lets users preview proposed and published changes

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • for loop used to operate on each element in an array
  • var keyword declares a variable
  • console.log method outputs a message to the web console by adding an argument. In this case console.log will output the value of topics to the web console because we’ve added it as an argument inside the parentheses.
  • Control flow - the order in which a computer executes a code in script
  • In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page
  • if statement is a way of introducing decision making into our code
  • else if is a second conditional statement
  • Truthy
  • Array is a single variable that is used to hold a group of data
  • Strings need to be contained in quotation marks
  • Boolean values need no quotation marks
  • Index - a unique number assigned to individual data items to locate to data item in the array
  • Starting point for our for loop var x = 0;
  • Second statement is a condition, as long as the variable x is less than the length of the array, the loop will keep running. Use the length built in array property on our shapes array to express that condition x < shapes.length;
  • Final statement is what allows the array to iterate each time. Code x++ is the same as x+1 so every time the code loops, x will increase by 1
